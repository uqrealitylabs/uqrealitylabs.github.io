# Security Policy

## Reporting A Vulnerability

Do not open a public issue for vulnerabilities or accidental credential exposure.

Use GitHub private vulnerability reporting for this repository:

https://github.com/uqrealitylabs/uqrealitylabs.github.io/security/advisories/new

Include:

- affected commit or deployed page,
- reproduction steps,
- impact,
- whether credentials, private assets, or licensed files may be exposed.

## Project Scope

This website must not commit secrets, private assets, Cypress screenshots or
videos, generated build output, package caches, or local environment files.

## Maintainer Response

Maintainers should acknowledge reports, assess impact, rotate exposed
credentials if needed, and publish fixes or guidance when safe.
